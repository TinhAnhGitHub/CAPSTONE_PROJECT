
# Agentic Video Deep Search ‚Äî Index‚ÄëThen‚ÄëAct, Tool‚ÄëCentric Design

## Abstract
We present an agent‚Äëcentric system for answering natural‚Äëlanguage queries about long‚Äëform videos. Instead of prompting a model end‚Äëto‚Äëend, we first index videos into coherent instances‚Äîsegments, frames, captions, and embeddings‚Äîthen expose them as typed tools that agents can plan over. This index‚Äëthen‚Äëact separation makes behaviors more controllable, transparent, and reusable: ingestion builds the world; agents act in it.

## 1. Agent Perspective (Core)
The agent runtime treats every stored video artifact as a first‚Äëclass tool input/output. Tools are registered with categories, tags, dependency contracts, and structured return types; a factory binds runtime clients (Milvus, MinIO, Postgres, external encoders, LLM) behind those contracts. Agents then compose tool calls to retrieve evidence, navigate timelines, and synthesize answers.

- Registry and metadata
  - Central `ToolRegistry` maintains names, categories, tags, dependency lists, type hints, and autogenerated docs.
  - Tool functions use explicit type annotations (Pydantic models for inputs/outputs), enabling safe planning and consistent formatting.

- Dependency injection
  - `ToolFactory` binds declared dependencies (e.g., `visual_milvus_client`, `postgres_client`, `minio_client`, `llm_as_tools`) so agents never see raw clients.
  - User‚Äëscoped context (e.g., `list_video_id`, `user_id`) is supplied at run time; infrastructure dependencies are injected by the factory.

- Output shaping
  - A `ToolOutputFormatter` converts domain objects (images, segments, videos, binary media) into LLM‚Äëfriendly content blocks‚Äîstructured JSON text for summaries; image/video payloads for visualization.

- Agent workflow
  - A multi‚Äëagent flow (greeting ‚Üí planner ‚Üí orchestrator ‚Üí workers ‚Üí consolidation) uses registry docs to propose and execute minimal tool sequences, streaming intermediate results and allowing human‚Äëin‚Äëthe‚Äëloop control when needed.

### 1.1 Tool Taxonomy
- Search
  - Visual text ‚Üí image (dense visual embeddings)
  - Caption text ‚Üí image (caption embeddings)
  - Multimodal (visual + caption dense + caption sparse hybrid ranking)
  - Event text ‚Üí segment (segment‚Äëlevel caption embeddings with hybrid support)
- Interaction / Navigation
  - Hop forward/backward among segments; list all segments for a video
  - Step through images temporally; extract ad‚Äëhoc frames by timestamp or window
- Utilities
  - Time/frame conversions; MinIO reads for images/videos
  - ASR context windows around a frame or a segment
- Prompt tools
  - Query enhancement (visual/textual) and new image captioning with optional ASR focus

### 1.2 Composition Patterns (Agent Playbook)
- Event QA
  1) `get_segments_from_event_query` with user intent and weights
  2) `get_related_asr_from_segment` to fetch local transcript context
  3) Optional: `extract_frames_by_time_window` to sample evidence frames
  4) Consolidate answer grounded in retrieved segments + ASR
- Visual similarity exploration
  1) `get_images_from_visual_query` (or `find_similar_images_from_image` if a reference frame exists)
  2) `get_images` to traverse neighbors along the timeline
  3) `caption_new_image` to focus the description on user intent
- Multimodal retrieval
  1) `enhance_visual_query` / `enhance_textual_query` to diversify intent
  2) `get_images_from_multimodal_query` with tuned weights
  3) Aggregate and rank by downstream scoring or task‚Äëspecific heuristics

## 2. Helper Substrate (Minimal, but Essential)
Ingestion is the world‚Äëbuilder: it transforms raw videos into artifacts and indices that tools operate over. Agents don‚Äôt manage this; they rely on its invariants.

- What ingestion guarantees
  - Aligned timelines: segments (autoshot) and ASR share a consistent time/frame basis
  - Durable evidence: originals, frames, and captions are addressable via stable S3‚Äëstyle URIs
  - Vector presence: images and segments have embeddings in Milvus for dense/sparse/hybrid search
  - Lineage: Postgres records parent/child relationships so tools can reconstruct context without recomputation

- Key artifact families
  - Video: original metadata (fps, extension)
  - Segments: shot boundaries with timestamps and frames
  - ASR: tokenized, time‚Äëaligned transcript
  - Images: extracted frames (representative or sampled)
  - Captions: segment and image descriptions
  - Embeddings: image dense; caption dense; caption sparse (hybrid search)

- Idempotence & hygiene
  - Ingestion uses stable IDs and existence checks to avoid duplicates in Milvus and storage.

## 3. Agent Runtime: Interfaces and Contracts
- Typed interfaces
  - `ImageObjectInterface`, `SegmentObjectInterface`, `VideoInterface` shape tool inputs/outputs‚Äîeach includes enough metadata to chain calls (e.g., `related_video_id`, `frame_index`, `timestamp`).
- Context parameters
  - `list_video_id` (scope) and `user_id` (tenant) constrain retrieval and route to the right collections/buckets.
- Dependencies (injected)
  - `visual_milvus_client`, `segment_milvus_client`, `postgres_client`, `minio_client`, `external_client`, `llm_as_tools`.
- Formatting
  - Search and navigation tools return structured JSON summaries; binary fetchers return media payloads (image/video) for direct rendering.

## 4. Multi‚ÄëAgent Orchestration (How Tools Become Answers)
- Greeting agent triages the request and decides whether to plan.
- Planner proposes a `WorkersPlan` referencing tool names, tasks, and limits; plan and rationale are streamable for inspection.
- Orchestrator assigns tool calls to workers; workers emit progress and partial answers.
- Consolidation agent synthesizes an answer, weaving retrieved segments, images, and ASR snippets into a coherent narrative.

This architecture makes tool availability and return types explicit to the planner‚Äîimproving predictability and making it easy to add new tools without breaking existing plans.

## 5. Operational Guidance for Agent Builders
- Always pass `list_video_id` and `user_id` to scope retrieval; let the factory inject infrastructure dependencies.
- Prefer hybrid search when possible (visual + caption dense/sparse); it stabilizes results across query styles.
- Use navigation tools (segment/image hops) to reduce redundant retrieval and promote structured reasoning.
- Include ASR snippets near frames/segments to boost grounding for text‚Äëforward questions.
- Re‚Äëcaption sampled frames with a focus prompt when the user intent is narrow or nuanced.

## 6. Limitations & Future Directions
- Domain shift: embeddings and captioners can underperform on certain video genres‚Äîmonitor and fine‚Äëtune when needed.
- ASR sensitivity: noisy audio/languages degrade subsequent caption/segment quality; consider confidence‚Äëaware heuristics.
- Freshness: ingestion is batch‚Äëoriented; online/near‚Äëreal‚Äëtime indexing is future work.
- Governance: track schema evolution and collection growth in Milvus; add retention/isolation policies for privacy.
- Learning to plan: incorporate learned rerankers and policy‚Äëaware tools to improve agent choices under uncertainty.

## 7. Conclusion
The index‚Äëthen‚Äëact paradigm, realized as a tool‚Äëcentric agent surface over video artifacts, reduces the gap between user intent and precise visual evidence. Ingestion constructs a reliable world; the agent navigates it with a compact, typed vocabulary of search, navigation, IO, and prompt tools. This separation yields controllable, extensible behaviors for multimodal video understanding.
